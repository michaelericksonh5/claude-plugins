Phase 2: Mesh Attachment Emission
Implement --mesh-plan mesh_plan.json.

Plan schema includes:

source layer
target slot
hull vertices
interior vertices
triangles
UVs
hull count
max vertex budget
Use trace_mesh_hull.py as input, not as final authority. Claude vision can propose mesh candidates, but deterministic tooling must validate geometry.

Validation:

vertices/UVs/triangles count consistency
triangle indices in range
hull has at least 3 vertices
no self-intersecting hull
atlas path exists
vertex count within H5G policy
Gate: one reviewed mesh fixture before generalizing.

Phase 3: Weighted Mesh Emission
Implement --auto-bind-weights only when a mesh plan exists.

Use auto_bind_weights.py to generate first-pass weights, then validate:

every vertex has influence
bone indices exist
weights sum to 1.0
bones-per-vertex within policy
estimated vertex-transform cost recorded
Do not call this “final weight painting.” It is a first-pass automated binding that still needs animator review until deformation preview is accepted.

Final Definition Of Complete
The system is complete only when each feature has:

corpus evidence from past H5G games or reviewed fixtures
deterministic compiler implementation
schema/policy file
validator coverage
preview/runtime proof
eval coverage
docs updated from “helper-only” to “supported”
Until then, the skill should stay honest: useful, increasingly automated, but not magic.
